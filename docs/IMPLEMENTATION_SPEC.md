# 단어 학습 웹앱 — 구현 명세 (Claude Code용)

이 문서는 데스크탑 Claude Code에서 실제 구현을 이어가기 위한 명세다.
UX는 `vocab_quiz_mockup.html`(동작 프로토타입)에서 확정됐다. 그 파일이 시각·상호작용의 기준 레퍼런스다.

## 0. 무엇을 만드는가
- 자녀용 **영어 단어 카드 학습 웹앱**. 정적 사이트로 GitHub Pages 배포.
- 같은 데이터로 **인쇄용 지필 시험지**도 뽑는다(`paper.html`). 10절 참조.

## 1. 확정된 설계 결정 (변경 시 사용자 확인 필요)
- **호스팅**: GitHub Pages 정적. **백엔드 없음. API 키 없음.**
- **인증**: (A) 가림막 수준 — private 리포 + URL 비공개. 실제 접근통제 아님(정적 사이트 한계).
  - 주의: 무료 플랜은 private 리포로 Pages 배포 시 제약이 있을 수 있음(공개됨). 배포 시점에 확인.
- **판독은 앱에 넣지 않는다**: 사진→단어 판독·검증은 Claude 대화(게이트)에서 수행.
  산출된 words.json을 `scripts/import_words.py`로 변환해 리포에 커밋한다.
- **채점**: 자가채점(플립) 단일. 타이핑/필기인식 자동채점 없음(오판·복잡도 회피).
  손글씨 학습은 "종이에 쓰고 넘겨서 대조"로 유도(안내 문구).
- **저장**: 오답·진도는 **localStorage(기기별)**. 기기 간 동기화 없음(필요 시 후속 (C) Supabase).
- **데이터 구조**: DAY별 `data/dayNN.json` + `data/index.json`. 전량 보존.

## 2. 데이터 형식
### data/dayNN.json
```json
{ "id": 9, "title": "DAY 09  Clothes", "subtitle": "Clothes",
  "words": [ {"en":"fashion","ko":"명 유행, 패션"},
             {"en":"design","ko":"동 디자인하다, 설계하다; 명 디자인, 설계"} ] }
```
- `ko`는 **원자료 그대로** 저장한다(품사표시, 세미콜론, 번호 포함).
  표기 변환(박스·개행)은 **표시 단계에서만** 수행 — 데이터에 굳히지 않는다.
  - 이유: 표기 규칙이 바뀌어도 데이터를 다시 만들 필요가 없다. 실제로 2026-07-21에
    품사 문자를 4종 → 11종으로 늘렸는데 `data/*.json`은 한 글자도 건드리지 않았다.
  - 시험지(`paper.html`)는 `ko`를 **가공 없이 그대로** 인쇄한다(품사 박스 없음).

### data/index.json
```json
{ "days": [ {"id":10,"file":"day10.json","title":"DAY 10  School & Education","subtitle":"School & Education","count":30},
            {"id":9, "file":"day09.json", ...} ] }
```
- **id 내림차순(최신이 위)**. 앱은 이 순서를 그대로 신뢰한다.

## 3. 화면 사양 (모의 기준, 그대로 이식)
### 시작 화면
- 레이아웃: **고정 헤더 / 내부 스크롤 목록 / 고정 푸터**.
  - 목록만 자체 스크롤. 더보기로 늘어도 페이지가 화면 밖으로 밀리지 않는다.
  - 스크롤 위치에 따라 위/아래 **페이드 토글**: 맨 위면 상단 또렷, 맨 아래면 하단 또렷.
- DAY 목록: index.json 순서(최신 위). **기본 노출 2개**(`RECENT_VISIBLE=2`),
  나머지는 "이전 범위 더보기 (N개)"로 펼침/접힘.
- 기본 선택(체크): **최신 2개**(`RECENT_DEFAULT=2`).
- 푸터: **자동발음 켬/끔** 토글 + **학습 시작**(하단 고정, thumb zone).

### 퀴즈 화면
- 카드: 순서 셔플 + 방향 배정. 탭하면 뒤집혀 답 공개.
- **방향 규칙**(2026-07-21 개정): 세션 안에서 EK(영→한) : KE(한→영) = **5:5**(홀수면 ±1)를 강제하되,
  단어별로 **직전 세션의 반대 방향**을 우선 배정한다(`vq.dirHist`).
  - 이유: 방향마다 기르는 능력이 다르다(EK=뜻 알아보기, KE=영어 써내기). 매번 독립 추첨이면
    같은 단어가 연속 같은 방향으로 나올 확률이 2회 연속 50%·3회 연속 25%라 한쪽만 자란다.
  - 균형이 안 맞으면 이력 없는 카드부터 채우고, 그래도 모자라면 무작위로 골라 뒤집는다.
  - 이력 갱신은 **본 라운드만** — '틀린 것만 다시' 반복 라운드는 저장소를 건드리지 않는다.
- 채점: 답 확인 후 **알아요 / 틀렸어요**.
- 진행바 + `현재/총` 카운트. "그만" 시 진행 파기(저장 안 함).

### 결과 화면
- 정답률, 맞음/틀림 수, **틀린 단어 목록**.
- **틀린 것만 다시(맞출 때까지)**: 오답만 방향을 새로 배정해 재출제(`vq.dirHist` 는 갱신하지 않음).
  - 확정(9절): 오답이 남으면 자동으로 다음 라운드 반복, 전부 맞히면 종료(라운드 내 1회 정답 시 제외).
- 처음부터 / 범위 다시.

## 4. 표시 규칙 (뜻 렌더링) — 반드시 지킬 것
`ko` 문자열을 화면에 그릴 때:
1. **품사 박스**: 정규식 `(^|;\s*)(명|동|형|부|전|접|대|감|조|관|수)(?=\s)` 에 매칭되는
   품사표시를 테두리 박스로 렌더. (2026-07-21 개정 — 이전: `(^|[\s;])(명|동|형|부)(?=\s)`)
   - 문자열 내 **여러 개 모두** 변환. 구동사·숙어처럼 **없으면** 박스 없이 그대로.
   - 앵커가 **문자열 시작 또는 세미콜론 직후**로 좁혀져 있다. 문장 중간은 아예 검사하지 않으므로
     품사 문자를 11종까지 넉넉히 등재해도 "~할 수 있다"의 `수`, "전 세계"의 `전` 이 오탐되지 않는다.
   - 근거: 데이터 120단어 전수 검사에서 품사표시는 **예외 없이** 시작 또는 `;` 직후에만 나타났고,
     그 위치에 품사 아닌 글자가 온 사례는 0건이었다.
2. **품사 경계 개행**: **세미콜론(`;`)에서만** 줄을 나눠 품사별 블록으로 분리.
   - 번호 뜻(`1. … 2. …`)은 **한 줄 유지**(같은 품사 안의 여러 뜻이므로).
- 참고 구현(모의의 posHtml): 박스 치환 → `;` 기준 split → 각 조각을 `<span class="sense">`(display:block).

## 5. 발음 (Web Speech API)
- `speechSynthesis` 사용. **영어만** 발음(한글 뜻은 읽지 않음).
- 🔊 버튼은 **영어가 보이는 면**(앞/뒤 무관)에만 표시. 자동발음 설정과 무관하게 항상 동작.
- **자동발음 켬**: 영어가 보이는 면이 뜰 때 자동 발음(한글 면은 침묵).
- **iOS 제약**: 사용자 제스처 전에는 소리 불가. "학습 시작" 탭을 오디오 잠금 해제에 활용하되,
  사파리 버전에 따라 첫 카드가 침묵할 수 있음(그 경우 🔊 수동). 통제 밖 — 문서화만.
- 미지원 브라우저에서는 조용히 무시(에러 없이).
- **발음 속도 설정**(구현 추가): 천천히 0.3 / 보통 0.5(기본), 시작 화면에서 선택·`vq.settings.rate` 저장, 선택 시 미리듣기.
- **출제 신호음**(2026-07-21 추가): 한글이 앞면인 카드는 발음 대신 종소리를 낸다(자동발음 켬일 때만).
  Web Audio 합성이라 음원 파일이 없다. iOS 잠금 해제는 「학습 시작」·「오답 복습」 탭에서 `unlockAudio()`.
  - 채택음 **'반짝'**: C6·E6·G6(1047·1319·1568Hz)을 0.07초 간격으로 올려 치는 상승 3음.
    들리는 길이 약 0.5초, 최대 진폭 0.648. 음량은 `CHIME_VOL` 상수(1.0 = 발음과 같은 최대 진폭).
  - 선정 근거: ① 세션당 **15회** 울리므로 반복 내성이 최우선 → 짧을 것 ② 올라가는 음 = 시작·등장 신호
    (내려가는 음은 종료 신호라 부적합) ③ 1~2kHz = 소형 스피커가 실제로 낼 수 있는 대역.
  - 후보 비교·음량 조정 도구: `docs/chime-lab.html` (연속 5회 재생으로 반복 내성 확인 가능).

## 6. localStorage 저장 스키마 (구현 완료)
- 확정 키:
  - `vq.settings` → `{autoplay, rate}` (자동발음, 발음 속도 slow/normal)
  - `vq.wrong` → 누적 오답: `[{en, ko, day, missCount, lastSeen}]` (day = DAY id)
  - `vq.lastRange` → 마지막 선택 범위 `[id, ...]` (재방문 시 복원)
  - `vq.dirHist` → `{"en|dayId": "EK"|"KE"}` 단어별 직전 출제 방향 (세션 간 교대용, 3절)
  - `vq.paper` → 시험지 배정 이력 최대 2벌 `[{a:[{id, t1, o2}]}]` (10절)
    - `t1` = 시험지1의 방향 문자열(`"KEKEE…"`), `o2` = 시험지2의 행 순서.
      **단어 자체는 저장하지 않는다** — 매번 `data/*.json`에서 다시 읽어 재구성하므로
      4 DAY 한 벌이 546바이트다. 같은 정보를 두 곳에 두지 않기 위한 선택.
- 결정(9절): 오답은 **날짜 넘겨 누적**. 누적은 기기별로만 유효(동기화 아님).
- 저장소 갱신 규칙: **본 라운드(round 0)만 반영** —
  범위 학습 오답은 누적(missCount+1), 오답 복습은 첫 시도 정답 시 졸업(제거)·오답 시 missCount+1 유지.
  "맞출 때까지" 반복 라운드는 저장소에 영향 없음.

## 7. 캐시 무효화 (중요)
- GitHub Pages·브라우저 캐시가 옛 JSON을 물면 새 DAY가 늦게 뜬다.
- index.json / dayNN.json fetch 시 **버전 쿼리** 부여: 예 `fetch('data/index.json?v=' + Date.now())`
  또는 배포 해시. "커밋하면 자녀 기기에서 바로 반영"을 보장하려면 필수.

## 9. 미결 항목 → 결정 기록 (2026-07-14 확정)
- 오답 저장 범위: **날짜 누적** 채택(vq.wrong). 시작 화면에 "오답 복습 (N개)" 진입점 추가(0개면 숨김, 오프라인에서도 동작).
- "맞출 때까지" 반복 규칙: **자동 반복 + 1회 정답 졸업** 채택(오답이 남으면 자동 다음 라운드).
- 기본 노출·체크 수: 2 유지. `RECENT_VISIBLE`/`RECENT_DEFAULT` 상수(index.html 상단)로 노출 — 실사용 후 조정.
- 인증 게이트: **미도입 유지**(1절 (A) 가림막 수준 그대로). 트래픽 남용(DDoS 등)은 GitHub 인프라 영역·과금 없음 확인.

### 2026-07-21 확정
- **출제 방향**: 세션 5:5 유지 + **단어별 세션 간 교대** 도입(3절). 매번 독립 추첨은 한쪽 방향
  쏠림을 못 막는다는 판단. 대안이던 "현행 유지"·"한 세션에 양방향 2회 출제(문항 2배)"는 기각.
- **품사 정규식**: 문자 11종 확장 + 앵커 축소(4절). 문자만 늘리면 오탐이 생기므로 앵커를 함께 좁힘.
- **종소리**: '반짝'(상승 3음) 채택, `CHIME_VOL = 0.8`(5절). 하강형·긴 여운 후보는 기각.
- **시험지**: 웹앱에 내장(10절). 별도 PDF 생성 프로젝트와 **분리**하고 이 문서의 참조는 폐기.
  1단 배치만 구현, seed 없이 매번 새 무작위 + 직전 1벌 보관.
- (잔여 미결) DAY가 수십 개로 늘 때 그룹핑/검색 도입 여부 — 현재 미도입.
- (잔여 미결) 시험지 2단 배치(A4 한 장에 DAY 2개) — 종이를 절반으로 줄이고 싶어지면 추가.

## 10. 인쇄 시험지 (`paper.html`, 2026-07-21 추가)
- 진입: 시작 화면 「시험지 만들기 (인쇄용)」 → `paper.html?days=9,10` (범위를 쿼리로 전달)
- **배치**: DAY 하나당 A4 **한 장**, DAY마다 시험지1·2 두 장 → 총 `DAY수 × 2`장.
  순서는 `09-1 → 09-2 → 10-1 → 10-2` (DAY 오름차순).
- **출제**: 각 DAY 안에서 번호 1~30. 영어제시(뜻 빈칸) : 한글제시(영어 빈칸) = 정확히 5:5.
  시험지1은 **데이터 순서 유지·방향만 무작위**, 시험지2는 **방향 전부 반전 + 행 순서 재셔플**.
- **사양**(원본 파이썬 도구 `voca_pdf_tool.py` 의 `day` 레이아웃 이식): A4 세로, 여백 12mm,
  본문 10pt, 셀 padding 5.5pt/6pt, 격자 0.8pt, 열 폭 No 11mm · English 44mm · 뜻 나머지,
  표 헤더(`No/English/뜻 (한글)`)만 볼드, 정답 페이지·이름/점수 줄 없음.
- **무작위 확정 시점**: 페이지를 열 때와 「새로 뽑기」를 누를 때만. 인쇄를 다시 눌러도 화면과 같은
  시험지가 나온다. 「직전 시험지」로 한 벌 전으로 되돌릴 수 있다(인쇄 실패·재배부용).
- **인쇄 설정**: 사용자가 **A4 · 머리글/바닥글 끄기**를 지정해야 한다(CSS로 강제 불가). 화면에 안내 표기.
- ⚠️ **시험지2는 시험지1의 정답지다**(같은 단어, 방향만 반대). 두 장을 동시에 주면 시험이 성립하지
  않는다. 배치로는 막을 수 없으므로 **한 장씩 배부하는 운영으로 통제**한다.
- 실측(2026-07-21, DAY 09·10 · Chrome): 4장, 두 줄로 넘어간 행 0개, 세로 여유 8.4mm,
  뜻 칸 130.7mm에 최장 뜻이 82~92mm(폰트 무관) — 폰트가 바뀌어도 줄바꿈 여지 없음.
