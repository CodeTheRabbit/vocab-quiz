# 독해(Reading) 이미지 → `reading.json` 프롬프트

모바일 Claude 대화에 **한 UNIT의 어휘 리스트 사진(여러 장 가능)** 을 첨부하고, 아래 `---` 사이 내용을 그대로 붙여넣으세요.
받은 JSON을 리포 루트에 **`reading.json`** 으로 저장한 뒤:

```
python scripts/import_reading.py reading.json
```

→ `data/reading/unitNN.json` 저장 + `data/reading/index.json` 재생성(최신 UNIT이 위). 커밋·푸시하면 앱에 자동 반영됩니다.
**한 UNIT = JSON 한 개**입니다(사진 여러 장이 한 UNIT을 이룸).

---

첨부한 사진들은 독해 교재 **한 UNIT의 어휘 리스트**입니다(여러 장에 걸쳐 있을 수 있음).
표는 보통 **구분(지문) · # · 어휘(영어) · 뜻(한글)** 열입니다.

**묶는 방법:**
- 왼쪽 **구분** 열의 지문 제목(예: `1 Glass Harmonica`, `2 Salvador Dalí`)으로 단어를 **지문(passage)별로** 묶으세요. 지문 제목 앞의 번호는 떼고 이름만 (`Glass Harmonica`).
- 각 지문 안에서는 사진의 **번호(#) 순서 그대로** 나열하세요(섞지 말 것 — 앱이 순서대로 출제합니다).
- 사진의 모든 단어를 **빠짐없이** 포함.

**뜻(한글) 표기 규칙 — 사진 원자료를 그대로 보존하되 대괄호만 제거:**
- 품사 표기 `[명]`·`[동]` 등은 **대괄호를 떼고 「글자+공백」** 으로: `[명] 와인잔` → `명 와인잔`.
  품사 글자 11종: **명 동 형 부 전 접 대 감 조 관 수**.
- 한 단어에 **품사가 둘 이상**이면 세미콜론 `;` 로 구분: `[동] ~을 탓하다 [명] 탓` → `동 ~을 탓하다; 명 탓`.
- 품사 표기가 **없는** 구·숙어는 그대로: `유리그릇`, `편의점`, `돋보이다`, `~을 만들어 내다`.
- 한 품사 안에서 번호로 나뉜 여러 뜻(`1. … 2. …`)은 **한 줄로 그대로**.
- 쉼표·괄호·물결(~)·가운뎃점(·) 등 사진의 문장부호를 그대로 유지.

**정확도**: 철자·품사·뜻을 사진과 한 번 더 대조해 검증하세요. 흐릿해 확신이 안 서는 항목은 `⚠️` 로 표시해 되물어 주세요(추측 금지).

**출력 형식** (UNIT 번호·제목은 사진 상단/머리글에서 읽어 `unit` 에):

```json
{
  "unit": "UNIT 07  Arts",
  "passages": [
    {
      "title": "Glass Harmonica",
      "words": [
        ["wineglass", "명 와인잔"],
        ["blame", "동 ~을 탓하다; 명 탓"]
      ]
    },
    {
      "title": "Salvador Dalí",
      "words": [
        ["come up with", "~을 만들어 내다, (해답·아이디어를) 내놓다"],
        ["appealing", "형 매력적인"]
      ]
    }
  ]
}
```

먼저 **지문별 검증표**(# · 영어 · 뜻)를 보여주고, 그 아래에 **위 JSON만 코드블록으로** 출력하세요.
